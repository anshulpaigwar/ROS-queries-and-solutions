Useful Ubuntu Commands to remember:

Remove directory contents:

To remove the folder with all its contents(including all interior folders):

rm -rf /path/to/directory

To remove all the contents of the folder(including all interior folders) but not the folder itself:

rm -rf /path/to/directory/*

To remove all the "files" from inside a folder(not removing interior folders):

rm -f /path/to/directory/*

rm - stands for "remove"

-f - stands for "force" which is helpful when you don't want to be asked/prompted if you want to remove an archive, for example.

-r - stands for "recursive" which means that you want to go recursively down every folder and remove everything.

Copy all content from one directory to another:

You can copy the content of a folder /source to another existing folder /dest with the command

cp -a /source/. /dest/

Installations in ubuntu:

This is something very important that most of the people don't understand and just blindly copy paste the commands from websites to install Softwares and libraries in ubuntu. The easy installation process and linking of libraries actually make ubuntu superior to Windows and hence is the preferred choice of developers.

Well, there are two different type of installation using Pre-built Binaries or installation from source. You can find a detailed comparison between them here.

Installation using prebuilt binaries:

All software are programs, which are also called source packages. So all source packages need to be built first, to run on your system. The binary packages are one that are already build from source by someone with general features and parameters provided in the software so that a large number of users can install and use it.

1. APT:

APT (the Advanced Packaging Tool) is an evolution of the Debian .deb software packaging system. It is a rapid, practical, and efficient way to install packages on your system. Dependencies are managed automatically, configuration files are maintained, and upgrades and downgrades are handled carefully to ensure system stability.

Basically, it's the official command line tool provided and managed by team Ubuntu to help you install Softwares.

So what Softwares can I install through APT??

The of Softwares you can install using APT are limited. The list of Softwares/libraries are managed by team Ubuntu are chosen after rigorous testing that they won't affect your system stability. You can find the list by using following command:

$ apt-cache pkgnames

The list is stored in apt cache offline and you can search them. But you need internet connection to install them.

$ sudo apt-get update updates the apt cache list and version of Softwares you can install.

$ sudo apt-get install pkgnames command downloads packages from ubuntu servers and installs them.

$ sudo apt-get upgrade command actually compares the version of all the packages that are installed in your system with the version in apt cache and upgrades them if they are at the lower version.

I highly recommend you to go through these tutorials fo a better understanding: link-1, link-2.

Binary packages are ones which have been made specifically for one type of computer, or architecture. Ubuntu supports the x86 (i386 or i686), AMD64 and PPC architectures. By using APT correct binary packages will be used automatically, so you don't have to worry about picking the right ones.

2. Software center:

Software center is official GUI tool by team Ubuntu an alternative to APT command line tool for those who do not want to install software using command line. So you can search and install every damn library from software center that you can also do with APT.

3. aptitude:

Aptitude is another command line tool for installation and managing packages similar to apt-get with many added functionalities such as handling dependencies issues more smartly than apt-get.

4. Synaptic Package manager:

It is another GUI tool to install and manage your packages similar to software center but with some added functionalities such as finding and repairing broken packages.

5. Debian Packages:

What if you want to install a software that is not available in apt-cache??

Surely team Ubuntu cannot go through testing of all the available packages and hence there are limited number of packages in apt-cache. So developers whose packages could not it make to apt-cache use Debian package format(.deb) to pack their software for easy distribution and installation. Debian is standard UNIX archive. You can download .deb files from the websites of developers.

dpkg is a low-level system tool to extract, analyze, unpack and install or remove .deb files.

$ sudo dpkg -i pkgname.deb command installs the package.

For more information please go through this tutorial.

All the above mentioned package mangers are advanced interfaces to the Debian packaging system.

Key point: The package installed directly using dpkg in not managed by apt-get i.e if a new version of the package is available it will not be upgraded using $ sudo apt-get upgrade unless you add it to your apt repository. A detailed description about this can be found here.

Installation from Source:

Unlike pre-built binaries the Source packages are not pre compiled So for installing from source, you need to build the source code yourself. That means you need to take care of the dependencies yourself. Also you need to be aware of all the features of the package so that you can build it accordingly. Advantages of installing form from source are:

  • You can install the latest version and can always stay updated!
  • Install it in a location you wish.

The source code for software on Linux comes in the form of compressed tar files, which typically have either .tar.gz or .tar.bz2 extensions. Download the tar files of packages you wanna install and extract them using following command:

$ tar -xvfz <name of tarball with .tar.gz extension>
(or)
$ tar -xvfj <name of tarball with tar.bz2 extension>

Always read the Readme file provided with the package before installation. Readme file generally have valuable information about package dependencies and the steps for compillation and installation of the packages. Usually developers use makefile system to help you build and install the package. Steps involved are as follows:

$ cd pkgname get inside the extracted pkg directory

$ ./configure This command executes the configuration file which generates makefile according to the information in the file. You can edit configuration file to compile the source code according to your requirement. You may also find that makefile in the package already exist then this process would become unnecessary.

$ make command compiles the source code.

$ make install command installs the package into your system.

For more detailed information on how to install from source please see these tutorials: link-1, Link-2.

Compile, Build and Run Code!

To make a successfully working peice of software you must understand all the process involed to run your code:

Build vs compile:

  • Compiling is the process of converting the high level code to machine level code (object code).

  • Linking is the act of combining object code with libraries into a raw executable.

  • Building is the process of converting the high level language to a executable. It would involve compiling and linking. Many compilers handle the linking step automatically after compiling source code.

There is no use in compiling alone as it does not produce the final excutable and hence you need to build always.

Compiler:

A compiler is a computer program (or a set of programs) that transforms source code written in a programming language (the source language) into another computer language (the target language), with the latter often having a binary form known as object code.The most common reason for converting source code is to create an executable program.

IDE(Integrated Development Environment):

An integrated development environment (IDE) is a software suite that consolidates the basic tools developers need to write and test software. Typically, an IDE contains a code editor, a compiler or interpreter and a debugger that the developer accesses through a single graphical user interface (GUI). Modern IDEs also have intelligent code completion and various other addons. Examples: Geany, Eclipse, NetBeans, Dev-C++, Codeblocks...

GCC, g++ and gcc:

  • GCC: GNU Compiler Collection, Referrers to all the different languages that are supported by the GNU compiler.

  • gcc: GNU C Compiler. gcc will compile: .c/.cpp files as C and C++ respectively.

  • g++: GNU C++ Compiler. g++ will compile: .c/.cpp files but they will all be treated as C++ files.

In simple words gcc and g++ are peice of software that converts c++ source code into executables through sequence of process compiling, linking and building. IDEs are simple graphical user interfaces that uses g++/gcc compiler to build your code. But you don't need an IDE to compile your code using g++/gcc you can do it through command lines in your terminal!

To compile your c++ code, use:

$ g++ foo.c -o bar

foo.c in the example is the name of the program to be compiled. This will produce an executable in the same directory called bar.out ( -o bar specifies the name of the output file) which you can run by typing this in your terminal:

$ ./bar.out

Make and CMake:

FAQ: We have got IDEs to build my source code. IDEs are very simple to use and have loads of features so what the heck Make and CMake are?? why they are even in existence??

ANS: Consider you are working on a big project comprises a lot of different files. Now you would have to build all the files everytime. Each file might require linking to different libraries. Keeping track of various dependencies while building each file one by one using IDE is practically very difficult.

Make and CMake are build automation tools that automatically builds executable programs and libraries from source code by reading files called Makefiles and CmakeLists repectively which specify how to derive the target program.

In easy words when working with large number of files you can create a simple list of (names/path) all the different files you want to build and their corresponding dependencies. Then you can use softwares like Make and CMake those read this list and builds all the listed source codes one by one using the given instructions. This means you now just have to call Make/CMake command in your package and everything will get automatically build.

For more detailed description about Make and CMake please follow this link

  • Make: Uses "makefile" to get knowledge of how to build the source code. The reason we need “Make” is because it enables the end user to build and install your package without knowing the details of how it’s done.

  • CMake: CMake stands for Cross-platform Make. It uses CMakeLists.txt file instead of makefile. For CMake, there are two steps: First, you need to setup your build environment. This creates a makefile or something equivalent depending upon the OS/platform you are using(linux or windows). Second, you perform the actual build in the selected build system. So CMake is better than Make!

Installation using Make:

$ ./configure checks if you have all the required dependencies and creates the Makefile.

$ make compiles the software using the rules of the Makefile.

$ make install moves the software to the correct location in the filesystem.

Installation using CMake:

Download and untar the package and check if it has CMakeLists.txt file in it. If yes then go inside the directory of downloaded package and follow these steps:

$ mkdir build make a new folder name "build" for the files generated after building your package.

$ cd build change directory to build folder.

$ cmake .. This commands tells CMake to build all the file in current directory using the source code in previous directory. ".." specifies to previous directory.
